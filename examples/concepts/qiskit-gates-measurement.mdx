## The mental shift

A circuit is a recipe, not a value. You **append** gates to a `QuantumCircuit`, and nothing happens until you run it on a backend. Measurement is the only operation with classical side effects — every gate before it is reversible. Get used to "build → run → measure" as the rhythm.

<LangTab lang="qiskit">

```python {3#construct 5-7#single-qubit 9#two-qubit 11#param 13-14#measure-all 17-18#explicit-measure} title="basics.py"
from qiskit import QuantumCircuit

qc = QuantumCircuit(3, 3)

qc.h(0)
qc.x(1)
qc.z(2)

qc.cx(0, 1)

qc.rx(0.7, 2)

qc.measure_all(add_bits=False)
print(qc)

# or measure specific qubits into specific classical bits:
qc.measure(0, 0)
qc.measure([1, 2], [1, 2])
```

<Note id="construct" kind="core" tip="QuantumCircuit(num_qubits, num_clbits)" remember="Two registers: qubits + clbits. Measurement is the bridge.">
The first arg is qubits, the second is classical bits (where measurements land). They're separate registers — measurement is the bridge from quantum to classical.
</Note>

<Note id="single-qubit" kind="core" tip="h, x, y, z, s, t, rx, ry, rz, p — the single-qubit gate menu" remember="Fixed gates: gate(qubit). Rotations: gate(theta, qubit). Always qubit LAST.">
`h(q)` is Hadamard (creates superposition). `x` is bit-flip, `z` is phase-flip, `y = ix*z`. `rx/ry/rz(theta, q)` are continuous rotations. All take qubit index(es) as the **last** argument(s).
</Note>

<Note id="two-qubit" kind="gotcha" tip="cx is CNOT — control, target order matters" remember="cx(control, target). ALWAYS control first, target second. Mixing them swaps your circuit.">
`cx(control, target)` flips the target if the control is |1⟩. This is the basic entangler. Other two-qubit gates: `cz`, `swap`, `cp(theta, c, t)`, `crx/cry/crz(theta, c, t)`. The order of the two indices is **always** (control, target) for controlled gates.
</Note>

<Detail id="two-qubit">
The argument order matters because CNOT isn't symmetric — the control and target play different roles. Worth seeing the difference explicitly:

```python
# Bell state: H on q0, then CNOT with q0 as control
qc1 = QuantumCircuit(2)
qc1.h(0)
qc1.cx(0, 1)   # entangles: |00⟩ + |11⟩

# Different result: CNOT with q1 as control acts on Hadamard'd q0 differently
qc2 = QuantumCircuit(2)
qc2.h(0)
qc2.cx(1, 0)   # NOT a Bell state — q1 is |0⟩, never flips q0
```

CNOT decomposition trick — symmetric controlled-Z is sandwich-able:

```python
qc.h(target)
qc.cz(control, target)   # cz IS symmetric: cz(a,b) == cz(b,a)
qc.h(target)
# equivalent to qc.cx(control, target)
```

Other two-qubit gates and their argument conventions:

- `swap(q0, q1)` — symmetric, order doesn't matter
- `cz(c, t)` — symmetric mathematically, but Qiskit still requires (control, target) for transpilation
- `cp(theta, c, t)` — controlled phase: target gets `e^{i*theta}` only if control is |1⟩
- `crx/cry/crz(theta, c, t)` — controlled rotation, theta first per the rotation convention

If your Bell-state results come out as 50/50 between '00' and '01' instead of '00' and '11', you almost certainly swapped a CNOT's control and target.
</Detail>

<Note id="param" kind="mistake" tip="Continuous rotations take theta first, qubit second" remember="rx(theta, qubit). NOT rx(qubit, theta). Most common Qiskit signature bug.">
A common bug: writing `qc.rx(0, 0.7)` (qubit then angle). The signature is `rx(theta, qubit)`. Match the math notation: the angle is the parameter, the qubit is the operand.
</Note>

<Note id="measure-all" kind="mistake" tip="measure_all(add_bits=False) reuses the existing classical register" remember="add_bits=False or Qiskit silently appends a NEW 'meas' register. Breaks cregs[0] downstream.">
`add_bits=False` reuses the existing classical register. Without that flag, Qiskit silently appends a new classical register named `meas`, which trips up downstream code that expects `qc.cregs[0]`.
</Note>

<Note id="explicit-measure" kind="core" tip="measure(qubit_or_list, clbit_or_list)" remember="Unmeasured qubits = discarded at end. Want it back? Measure it.">
Use this when you only care about specific qubits, or when you want to map qubits to classical bits in a specific order. Anything not measured stays in superposition (and is discarded when the simulator finishes).
</Note>
