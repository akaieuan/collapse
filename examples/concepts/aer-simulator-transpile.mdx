## The mental shift

Qiskit is layered: you build a high-level circuit using whatever gates are convenient, but **backends** (simulators or hardware) only support a limited native gate set with specific qubit connectivity. `transpile` rewrites your circuit to fit the backend. Skip transpile and most real backends will reject your circuit; on Aer, things mostly work but you miss the optimization passes.

<LangTab lang="qiskit">

```python {1-2#imports 4-7#circuit 9-10#sim 12#transpile 14#run 15-16#counts 18#histogram} title="run_basics.py"
from qiskit import QuantumCircuit, transpile
from qiskit_aer import AerSimulator

qc = QuantumCircuit(2, 2)
qc.h(0)
qc.cx(0, 1)
qc.measure([0, 1], [0, 1])

sim = AerSimulator()
# sim = AerSimulator(method="statevector")  # exact, no shots
# sim = AerSimulator(noise_model=...)        # realistic noise

transpiled = transpile(qc, sim, optimization_level=2)

job = sim.run(transpiled, shots=4096)
result = job.result()
counts = result.get_counts()

print(counts)  # e.g. {'00': 2031, '11': 2065}  — Bell pair
```

<Note id="imports" kind="mistake" tip="qiskit.transpile + qiskit_aer.AerSimulator are the two anchors" remember="Aer ships SEPARATELY from qiskit. pip install qiskit-aer.">
`qiskit-aer` is a separate package. If you see `ModuleNotFoundError: qiskit_aer`, run `pip install qiskit-aer`. The base `qiskit` package no longer includes Aer in modern versions.
</Note>

<Note id="circuit" kind="core" tip="A 2-qubit Bell circuit is the canonical 'hello world'" remember="h(0) + cx(0,1) = Bell state. Always 50/50 between '00' and '11'.">
`h(0)` + `cx(0, 1)` produces the Bell state (|00⟩ + |11⟩)/√2. After measurement, you'll see ~50% '00' and ~50% '11' — never '01' or '10' (modulo simulator noise).
</Note>

<Note id="sim" kind="note" tip="AerSimulator() defaults to method='automatic'" remember="method='statevector' for exact, no-shots simulation. Default = let Aer pick.">
For exact statevector access (no shots, no sampling), pass `method='statevector'` and use `result.get_statevector()`. For density-matrix work, `method='density_matrix'`. For noisy simulation, build a `NoiseModel` and pass `noise_model=...`.
</Note>

<Note id="transpile" kind="core" tip="optimization_level: 0=fastest compile, 3=best circuit" remember="Aer: level 1. Hardware: level 2-3. Always set seed_transpiler when tuning.">
For Aer, level 1 is a good default. For real hardware, always use level 2 or 3 — the gate count reduction matters because of decoherence. Pass `seed_transpiler=42` for reproducibility while tuning.
</Note>

<Note id="run" kind="core" tip="shots is the most important runtime knob" remember="More shots = smoother stats, slower run. Default 1024.">
More shots = better statistics, slower runtime. 1024 is the IBM Quantum default. For probability-amplitude reconstruction (like quantum-audio's QPAM), more shots = smoother decoded waveform.
</Note>

<Note id="counts" kind="mistake" tip="counts is a dict of bitstring → frequency" remember="Bitstrings read RIGHT-TO-LEFT by qubit index. '01' = q0:1, q1:0. Mirrored from how you'd intuit.">
The bitstring is read **right-to-left** by qubit index — `'01'` means qubit 0 measured 1, qubit 1 measured 0. This trips up basically everyone the first time. If your results look mirrored, that's why.
</Note>

<Detail id="counts">
The reason: Qiskit follows the physics convention where the most-significant bit is the highest-indexed qubit. So a 3-qubit measurement returns `'q2 q1 q0'`, not `'q0 q1 q2'`.

Worked example. A circuit that puts |1⟩ on qubit 0 and leaves the others at |0⟩:

```python
qc = QuantumCircuit(3, 3)
qc.x(0)
qc.measure([0, 1, 2], [0, 1, 2])
counts = AerSimulator().run(qc, shots=1024).result().get_counts()
# counts == {'001': 1024}   — that '1' is qubit 0, on the right
```

Three ways to deal with it:

- **Read right-to-left**: train yourself to map `'001'` → q2:0, q1:0, q0:1. Cleanest if you only do quantum work.
- **Reverse before parsing**: `int(bitstring[::-1], 2)` to convert to a qubit-index-ordered integer.
- **Use `marginal_counts`**: when you only care about a subset of qubits, this picks them out without you having to do bitstring math.

```python
from qiskit.result import marginal_counts
just_q0 = marginal_counts(counts, indices=[0])
# {'0': N0, '1': N1} — no ordering ambiguity
```

The right-to-left convention is also why `qc.cregs[0]` is the LOW-order register in the bitstring. If you append a second classical register later, those bits become the HIGH bits.
</Detail>

<Note id="histogram" kind="note" tip="From counts, you can plot, decode, or compute expectation values" remember="plot_histogram(counts) for matplotlib. counts.items() for code.">
For visualization, `from qiskit.visualization import plot_histogram; plot_histogram(counts)` returns a matplotlib figure. For numerical post-processing, just iterate `counts.items()`.
</Note>
